5. Create Custom Connector

LAB SCENARIO

Working as part of the PrioritZ fusion team you will be configuring a custom connector for a new API you build using Azure Functions. The team has decided to move the logic when a user creates a new “ask” to the Azure Function API. This will keep the Power App formula simple and allow more complex logic to be added in the future. In this lab you will create the function, use the Dataverse API, secure the API with Azure AD, configure a custom connector to use the API, and change the Power App to use the connector.

Note: This lab requires an Azure subscription (or trial) in the same tenant as your Dataverse environment.

In Exercise 5 you will create a new custom connector.

Important

This lab requires an 🅰️Azure subscription (or trial) in the same tenant as your Dataverse environment.

5.1 Create custom connector​

  1. Navigate to Azure Portal
  2. Select All resources, search for the function app you deployed and click to open it.

  1. Copy the URL. Open a new browser tab or window and paste the URL you copied.

  1. Add /api/swagger.json to the end of the URL and ENTER. Click "Accept" if prompted for permissions.

  1. Right click on the swagger and select Save as.

  1. Save the file on your local machine.

  2. Navigate to Power Apps maker portal and make sure you have the correct Dev environment selected.

  3. Expand Dataverse and select Custom Connectors.

  1. Click on the chevron button next to the New custom connector and select Import an OpenAPI file.

  1. Enter PrioritZ Connector for name and click Import.

  1. Select the swagger file you saved and click Open. 👉 Then click Continue.

  2. Provide Description and advance to Security.

  1. Select OAuth 2.0 for Authentication type.
  2. Select Azure Active Directory Identity provider.
  3. Copy the PrioritZFL Connector application ID from your notepad and paste it in the Client id field.
  4. Copy the PrioritZFL Connector Secret your notepad and paste it in the Client secret field.
  5. Copy the Tenant ID from your notepad and replace common with it in the Tenant ID field.
  6. Copy the PrioritZ API application ID from your notepad and paste it in the Resource URL field.
  7. Enter true for Enable on-behalf-of login.
  8. Click Create connector.

5.2 Test connector​

  1. Select the Test tab. 👉 Click + New connection. 👉 Click Create.

  1. Provide your credentials. 👉 Click Accept. 👉 Then Select the Test tab.
  2. Select Custom connectors and click Edit on the PrioritZ connector.

Note

Make sure the connection you created is selected.

  1. Turn on Raw Body. Provide the JSON below and click Test operation.
